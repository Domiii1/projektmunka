Vissza a Projektekhez

Kis- és Közepes Vállalati Hálózat Tervezése és Adatbázis Implementálás

1. Bevezetés

Ez a dokumentáció bemutatja egy kis- és közepes méretű vállalat számára készült hálózati infrastruktúra tervezését, megvalósítását IPv4 címzéssel, valamint egy egyszerű egytáblás adatbázis tervezését és implementálását.

2. Kis- és Közepes Vállalati Hálózat Tervezése IPv4 Címzéssel

2.1. Hálózati igények és célok

A vállalat számára szükséges egy stabil, gyors és biztonságos hálózati infrastruktúra, amely képes kiszolgálni a mindennapi üzleti műveleteket. Az IPv4 címzés használatával történik a hálózati eszközök azonosítása.

2.2. IPv4 Címzési Tervezés

A hálózati címek tervezéséhez figyelembe kell venni a vállalat eszközeinek számát, a különböző alhálózatokat és a hálózaton belüli kommunikációt. A tervezéshez a következő címzés javasolt:

192.168.0.0/24

Ez biztosítja a vállalat összes eszközének azonosítását egy 254 eszközig. Az alhálózatokat szükség szerint tovább lehet bontani.

3. Egytáblás Adatbázis Tervezése

3.1. Adatbázis célja

A cél egy egyszerű, egytáblás adatbázis létrehozása, amely tárolja a vállalat munkatársainak adatait.

3.2. Adatbázis szerkezete

A táblázatnak a következő mezőket kell tartalmaznia:

  • ID: Egyedi azonosító
  • Vezetéknév: A munkavállaló vezetékneve
  • Keresztnév: A munkavállaló keresztneve
  • Email: A munkavállaló email címe
  • Telefonszám: A munkavállaló telefonszáma

3.3. SQL Táblázat létrehozása


CREATE TABLE Munkavallalok (
    id INT PRIMARY KEY AUTO_INCREMENT,
    vezeteknev VARCHAR(50),
    keresztnev VARCHAR(50),
    email VARCHAR(100),
    telefonszam VARCHAR(20)
);

3.4. Módosítás és Adatok Lekérdezése

Az adatok módosítására és lekérdezésére az alábbi SQL parancsokat használhatjuk:


-- Új munkavállaló hozzáadása
INSERT INTO Munkavallalok (vezeteknev, keresztnev, email, telefonszam)
VALUES ('Kovács', 'Anna', 'anna.kovacs@example.com', '06201234567');

-- Munkavállaló adatainak módosítása
UPDATE Munkavallalok
SET email = 'anna.newemail@example.com'
WHERE id = 1;

-- Munkavállaló adatainak lekérdezése
SELECT * FROM Munkavallalok WHERE id = 1;

4. Összegzés

Az IPv4 alapú hálózati tervezés biztosítja a vállalat számára szükséges eszközkészlet és alhálózati struktúra kialakítását, míg az egytáblás adatbázis lehetőséget biztosít az adatok hatékony kezelésére és lekérdezésére. A megfelelő tervezés és kivitelezés hozzájárul a vállalati rendszer zavartalan működéséhez.
